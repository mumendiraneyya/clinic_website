# موقع عيادة د. مؤمن ديرانية

![لقطة شاشة للموقع](screenshot.png)

## نظرة عامة

موقع طبي مبني على إطار عمل [Astro](https://astro.build/) مع قالب AstroWind. يدعم اللغة العربية (RTL) ويتضمن:

- صفحة رئيسية تعريفية
- مدونة للمقالات الطبية
- نظام حجز مواعيد عبر Cal.com
- خريطة وموقع العيادة

## التشغيل

```bash
npm install              # تثبيت المتطلبات
npm run dev              # تشغيل خادم التطوير على localhost:4321
npm run claude           # يشغل كلاود أن كنت ترغب بالتطوير باستخدام الذكاء الاصطناعي
```

## تحديث الموقع

لتحديث الموقع:

```bash 
npm run deploy           # بناء ونشر الموقع على abuobaydatajjarrah.com
```

### وضع الصيانة

لنشر الموقع مع تعطيل الحجز مؤقتًا:

```bash
npm run deploy:maintenance
```

السطر أعلاه ينشر الموقع ويعطل زر الحجز مؤقتًا، فهو معد لاستخدامه أثناء فترات الصيانة. توجد أيضًا طريقة أخرى لفعل الشيء نفسه وإنما بشكل أكثر ديمومة وهي بتغيير إعداد الموقع المسمى bookingDisabled في أعدادات القالب من [هنا](./website/src/config.yaml)

انتبه أن تغيير إعدادات القالب سوف يعطل عملية الحجز بغض النظر عن أمر npm المستخدم أعلاه

### نشر إصدارة معدلة من الموقع

في عالم التطوير يدعى هذا بالتجهيز (Staging) وهو ممكن باستخدام تعليمة النشر ذاتها أعلاه٬ مع فرق بسيط٬ وهو أنك حينما تنفذ هده التعليمة تكون على فرع غير الفرع الأساسي `main`.

مثلاً، لنفترض أنك تعمل على فرع اسمه `seo` وحاولت نشر الموقع من هناك. ما سيحصل هو أنك ستنشر إصدارة تجهيزية على العنوان:
> https://seo.abuobaydatajjarrah.pages.dev

علمًا بأن الموقع المنشور على https://abuobaydatajjarrah.com لن يتأثر في هذه الحالة
